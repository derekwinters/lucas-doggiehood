Why

The three wastes that eat delivery

Translation waste

Turning an idea into work a developer can start: requirements churn, follow-up questions, re-litigated scope.

Readiness drift

"Ready" means something different on every team and every story — so sprints absorb work that isn't buildable.

Idle capacity

Decisions queue behind calendars. And between 6pm and 9am, nothing happens at all.

None of these are coding problems. They're pipeline problems.

Ship

The morning after: one reviewable PR

  • Opens with a mandatory Deviations & Decisions section — the agent self-reports anything off-plan, including nothing.
  • Lists built / dropped / deferred — a truncated run never reads as complete.
  • An engineer reviews against the acceptance criteria a human already approved, and merges by hand.
  • Your CI, branch protection, and release tooling — unchanged.

Trust

Safe to run unattended — by construction

  • Authorization — only allowlisted roles move work; ignored commands are reported, not just dropped.
  • Idempotency — processed comments are watermarked; re-runs with nothing new change nothing.
  • No invented requirements — the AI stops and asks; /propose is the only path to a draft.
  • No self-merge — branch protection makes it a platform guarantee.
  • Tested guardrails — parser, queue, dashboard: plain scripts with unit tests in CI.

Cadence

It runs on a clock, not on attention

Overnight

Gatekeeper sweeps commands, then the TDD build run — one PR per opted-in repo by morning.

Morning

Research pass on newly admitted work; the daily digest lists exactly what awaits whose decision.

Evening

Command sweep — the day's approvals and revisions are applied before the night's build.

Type a command at any hour, from any device. The next routine picks it up.
