# Pipeline dashboard

Read-only reporter. A **script** (`render_dashboard.py`) recomputes live state
and rewrites the body of issue **#193** in place. There is no LLM generation of
the body — that saves tokens and guarantees a byte-stable format. See
`docs/engineering/issue-pipeline.md`.

## How it runs in production (scheduled workflow — not an AI routine)

The dashboard is generated by **`.github/workflows/dashboard.yml`**, a
scheduled GitHub Actions workflow, *not* by an AI routine shelling out. This
was a deliberate pivot from the original epic design: the render is fully
deterministic and needs no model, so a cron workflow is cheaper, more reliable,
and — crucially — authenticates the headless PATCH with the built-in
`GITHUB_TOKEN` (which resolves the auth question raised in #196; no extra
secret is needed). The AI routines (gatekeeper → analysis, gatekeeper → dev)
therefore no longer include a dashboard step.

The workflow runs `render_dashboard.py --write`, which:

1. `fetch_state()` — queries the GitHub REST API (stdlib `urllib`, no pip
   deps) for issues by label, milestone roll-ups, open PRs (release-please
   broken out), the focus marker on #193, and blocked state.
2. `render_body(state)` — renders the approved markdown (see below).
3. PATCHes **only** #193's body. Mutates nothing else.

## What it renders

Matches the approved format in #193: the hidden `<!-- pipeline-focus: ... -->`
marker (first line — the shared focus source of truth), the focus-milestone pie
(green done / yellow ready-for-work / red remaining), the focus ready-for-work
queue (blockers flagged), the "Your move" counts, PRs, intake, pending-approval,
needs-clarification, other-milestone progress bars, the open-issues bar chart,
and the command reference. Issue **titles** are used as the summary text
(deterministic — no model), which is the tradeoff for a fully-scripted body.

`#193` itself and any `parked` issue are excluded from every queue.

## Manual / preview use

- Preview from a fixture (no network): `render_dashboard.py --render-only <
  state.json`.
- Refresh live from a machine with a token:
  `GITHUB_TOKEN=... python3 render_dashboard.py --write`.

## Tests

`tests/test_render.py` checks structural invariants (pie values and colors,
focus marker, section headers, exclusions) and a **golden snapshot**
(`tests/expected_dashboard.md`) rendered from `tests/fixture_state.json`, so any
format drift fails CI. Run:

```bash
python3 -m unittest discover -s .claude/skills/pipeline-dashboard/tests
```

To intentionally change the format: edit the renderer, regenerate the golden
with `render_dashboard.py --render-only < tests/fixture_state.json >
tests/expected_dashboard.md`, and review the diff.
